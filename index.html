… is a CSS library to make attractive documentation for software projects. Focus is placed on simplicity & clarity with a nod to the monospaced world that we live in while working.

It is perfectly suited to styling Markdown-generated documents, although manually-authored man-css HTML will provide some capabilities that are not supported in Markdown’s simplified syntax/structure.

#Usage

Simply include the stylesheet in your page…

<link rel="stylesheet" type="text/css" href="http://davidfmiller.github.io/man-css/man.css">

… or save the file locally within your project’s repository and include it from there.

This page is built using man-css; save and copy the source to use as a template for your project’s documentation.

Code blocks

… should be placed within <pre></pre> tags. Output from pygments HTMLFormatter will receive syntax highlighting, ex: 

<!DOCTYPE html>
<html lang="en">
<head>
  <title>man-css</title>
  <meta charset="utf-8">
  <meta name="viewport" value="width=960">
  <link rel="stylesheet" type="text/css" href="man.css">
</head>
<body>
<-- markup -->
</body>
</html>#

Text Selection

Use an <i> tag within a <pre> block to prevent text-selection on elements (ex: command prompts, comments, etc.)
